HTML | How I
Structure Web Pages

HTML is the foundation of every page I build. I always write semantic HTML5, using the correct element for each purpose. This improves accessibility, helps search engines understand my pages and makes my code readable for other developers.

❌ How I used to write HTML

Before learning semantics, everything was just divs with no real meaning.

<!-- Non-semantic: everything is <div> --> <div id="top"> <div class="menu"> <div class="item">Home</div> </div> </div> <div id="wrap"> <div class="title">Hello</div> </div>

✓ How I now write HTML

Semantic elements give clear meaning to browsers, screen readers and search engines.

<!-- Semantic HTML5: clear purpose --> <header> <nav> <a href="/">Home</a> </nav> </header> <main> <article> <h1>Hello</h1> </article> </main>

My Project Boilerplate

Every project I start uses this structure. The viewport meta tag, linked stylesheet and semantic body elements are always there from the beginning.

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>My Project</title> <link rel="stylesheet" href="style.css"> </head> <body> <header><nav><ul><li><a href="#">Home</a></li></ul></nav></header> <main> <section> <h1>Welcome to My Portfolio</h1> <article> <h2>About Me</h2> <p>I am Aashika, a BSc Computer System Engineering student...</p> <img src="profile.jpg" alt="Aashika Dallakoti"> </article> </section> </main> <footer><p>&copy; 2026 Aashika Dallakoti</p></footer> <script src="script.js"></script> </body> </html>

Forms I Build

I always link labels to inputs using for and id. Try clicking Submit with empty or invalid data!

<form> <label for="name">Full Name</label> <input type="text" id="name" required> <label for="email">Email</label> <input type="email" id="email" required> <button type="submit">Submit</button> </form>

Semantic Elements I Use in Every Project

<header>

Logo, site title and navigation. Always at the top of my pages.

<nav>

Wraps navigation links. Screen readers identify this as a landmark.

<main>

The primary unique content of the page. One per page only.

<section>

Groups thematically related content with a heading.

<article>

Self-contained content such as blog posts, portfolio cards and news items.

<aside>

Supplementary content, sidebars or related links.

<figure>

Wraps images. I use <figcaption> for accessible captions.

<footer>

Copyright, contact links, secondary navigation at bottom.

<time>

Marks dates with a machine-readable datetime attribute.
